Assembly Data API¶

The assembly API can be used as a library or in client/server mode.

Creating an Assembly object¶

To create a new Assembly object, you will create an object that implements the interface defined in doekbase.data_api.sequence.assembly.api.AssemblyInterface. This class can implement a library or a Thrift client. In either case you will instantiate the class with an object reference. The interfaces used by both classes are shown below.

Python Library: AssemblyAPI¶

If you are using the Python library, then you can instantiate the doekbase.data_api.sequence.assembly.api.AssemblyAPI class. This will communicate with the back-end database directly. This is the mode used when you are creating code in Python code cells in the KBase Narrative.

__init__(services=None, token=None, ref=None)[source]¶

Construct a new AssemblyAPI object, which connects directly to the KBase backend stores.

Parameters:
  • services (dict) – Dictionary with service urls, including: workspace_service_url for the Workspace service, e.g. “https://kbase.us/services/ws/”.
  • token (str) – KBase login token
  • ref (str) – KBase object reference, in the format expected by the KBase workspace: workspace-id/object-id OR workspace-id/object-id/version, where the workspace and object can either be numeric identifiers or names. For example: 1019/4, 1019/4/1, or PrototypeReferenceGenomes/kb|g.166819.

Thrift Service: AssemblyClientAPI¶

If instead you are using Python or another language as a client, and running the AssemblyAPI within a Thrift server, you would use doekbase.data_api.sequence.assembly.api.AssemblyClientAPI in Python or the equivalent in the target language.
